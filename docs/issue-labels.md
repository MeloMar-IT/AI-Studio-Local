# Issue Labels

The following labels are used in the LTX Studio Local project to categorize and prioritize issues.

## Types
- `bug`: Something isn't working as expected.
- `feature`: New functionality or enhancement.
- `refactor`: Code change that neither fixes a bug nor adds a feature.
- `docs`: Documentation only changes.
- `chore`: Maintenance tasks, dependencies, build system, etc.
- `performance`: Improvements to speed or resource usage.
- `ui/ux`: Changes to the user interface or experience.

## Priority
- `priority:low`: Nice to have, but not urgent.
- `priority:medium`: Standard priority for most tasks.
- `priority:high`: Should be addressed soon.
- `priority:critical`: Blocks development or major functionality.

## Status
- `status:triage`: New issues waiting for review.
- `status:ready`: Confirmed and ready for implementation.
- `status:in-progress`: Currently being worked on.
- `status:blocked`: Cannot proceed due to external factors.
- `status:needs-info`: More information is needed from the reporter.

## Areas
- `area:app`: Issues related to the SwiftUI application.
- `area:worker`: Issues related to the Python worker.
- `area:shared`: Issues related to shared schemas or documentation.
- `area:ci-cd`: Issues related to GitHub Actions or automation.
